Extract Textual config wizard field ownership

Scope:
- Split Textual config wizard field metadata, Textual-safe input IDs, validation helpers, visible-field selection,
  value cloning, and default hydration into `ui/textual/screens/config_wizard_fields.py`.
- Move additional-service form parsing, integer validation, service rebuilding, and hidden-field preservation into the
  same field owner so the Textual screen only handles widget state, focus, and status output.
- Keep `ui/textual/screens/config_wizard.py` as the app/runtime screen owner and compatibility import surface for
  existing tests and hidden callers that import private wizard helpers.
- Update the structure guard and Python engine architecture inventory for the new Textual wizard owner.

Key behavior changes:
- Preserve `AppServiceConfig.enable_if_path` when hydrating wizard values.
- Preserve hidden additional-service fields such as `startup_group` and `enable_if_path` when the wizard edits the
  visible fields for an existing additional service.
- Additional-service form validation now has direct owner tests for blank slug removal, invalid port focus metadata,
  dependency normalization, and hidden-field retention.
- No intended changes to visible wizard flow, Textual rendering, selectable-row styling, config persistence output, or
  command/test suggestions.
