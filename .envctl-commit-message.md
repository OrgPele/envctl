Centralize action target identity handling

Scope:
- Add typed action target identity helpers in `action_target_support.py` for normalized project names and roots.
- Route action environment/replacement construction, test target context construction, test plan status, test-summary
  project roots, migrate result summaries, and worktree delete/self-destruct flows through the shared target identity
  boundary.
- Preserve name-only status behavior for call paths that legitimately receive targets without roots while requiring
  roots for file-system-sensitive action identities.

Key behavior changes:
- No intended CLI behavior change for valid action targets.
- Invalid or rootless targets are now skipped consistently in root-sensitive action paths instead of each owner
  reimplementing name/root extraction differently.
