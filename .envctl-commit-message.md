Extract Textual config wizard hint ownership

- Move config wizard field hints, detected-suggestion matching, existing `.envctl` value labeling, and
  directory/entrypoint validation policy into `ui/textual/screens/config_wizard_hints.py`.
- Introduce `ConfigWizardHintResolver` so the Textual screen delegates field policy through a testable owner object
  while keeping legacy private screen methods as compatibility wrappers.
- Add direct resolver tests for existing-value hints, matching test-command suggestions, frontend test-path hints,
  and directory/entrypoint validation semantics.
- Update the structure guard and Python engine architecture inventory for the new hint owner.
