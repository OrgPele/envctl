fix(config): stop silent global ignore bootstrap during envctl config save

- Scope:
  - stop interactive config saves from silently setting Git `core.excludesFile` when the user has no global excludes configured
  - surface explicit success-state messaging when envctl updates an existing global excludes file
  - lock the warning-only missing-config contract with regression coverage across persistence, headless config output, and wizard save messaging

- Key behavior changes:
  - `python/envctl_engine/config/persistence.py`
    - changed `ensure_global_ignore_status(..., update_config=True)` so a missing `core.excludesFile` now returns the existing `missing_global_excludes_configuration` warning instead of auto-configuring a new global excludes file
    - added `ignore_status_summary(...)` so config surfaces can report updated/already-present global-ignore outcomes consistently
  - `python/envctl_engine/config/wizard_domain.py`
    - updated `_save_message(...)` to append rendered global-ignore success summaries for existing excludes targets while preserving warning-path hyperlink rendering
  - `python/envctl_engine/config/command_support.py`
    - updated non-JSON `envctl config` output to print the global excludes target when envctl updates an existing excludes file
  - `python/envctl_engine/ui/textual/screens/config_wizard.py`
    - switched the save path back to warning-only global-ignore handling instead of attempting interactive bootstrap
  - `tests/python/config/test_config_persistence.py`
    - added a regression proving `save_local_config_with_ignore_policy(..., update_global_ignores=True)` does not set `core.excludesFile` when it is missing
  - `tests/python/config/test_config_command_support.py`
    - added plain-output coverage for the updated existing-global-excludes message
  - `tests/python/config/test_config_wizard_domain.py`
