# Envctl Prompt Overwrite Confirmation And Origin Review Preset

## What changed
- Replaced `install-prompts` backup-on-overwrite behavior (`.bak-*` file creation) with a single aggregated overwrite confirmation prompt
- Added new built-in `review_worktree_imp` prompt preset for origin-side review of implementation worktrees
- Updated user docs to document overwrite confirmation flow and new preset inventory
- Extended test coverage for overwrite approval, decline, JSON/non-TTY failure modes, and new preset discovery

## Key behavior changes
- `install-prompts` now prompts once for all overwrite candidates instead of creating timestamped `.bak-*` backup files
- `--yes` and `--force` bypass the interactive prompt for automation-safe overwrites
- `--json` mode + non-TTY runs fail cleanly with actionable error when overwrite approval is required but not pre-approved
- `--dry-run` does not trigger overwrite prompting and reports planned status
- New `review_worktree_imp` preset treats current local repo as unedited baseline and target worktree as edited implementation
- Review preset is read-only, findings-first, and accepts `$ARGUMENTS` for worktree path override and reviewer notes

## File paths/modules touched
- `python/envctl_engine/runtime/prompt_install_support.py` - core overwrite confirmation flow
- `python/envctl_engine/runtime/prompt_templates/review_worktree_imp.md` - new origin-review preset template
- `docs/user/ai-playbooks.md` - updated overwrite docs + new preset usage note
- `docs/reference/commands.md` - updated AI CLI presets section
- `docs/user/python-engine-guide.md` - updated install-prompts behavior note
- `tests/python/runtime/test_prompt_install_support.py` - replaced backup assertions with confirmation/decline/JSON/non-TTY coverage
- `tests/python/runtime/test_command_exit_codes.py` - added overwrite-required exit code coverage

## Risks / notes
