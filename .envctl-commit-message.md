Extract config command default ownership

Scope:
- Move config-time command, directory, and frontend test-path default resolution into
  `config/command_defaults.py`.
- Keep `envctl_engine.config` as the import-compatible facade for existing private default-resolution names while
  removing direct action/runtime suggestion imports from the facade.
- Add focused tests for explicit value precedence, supported backend/frontend start-command detection, unsupported
  backend layout fallback, shared action-test command reuse, specific test-command overrides, and frontend test-path
  canonicalization.
- Update the structure guard and Python engine architecture inventory so future command/default resolution changes stay
  in the owner module instead of growing `config/__init__.py`.

Key behavior changes:
- No intended config semantics changed; `load_config` and existing private compatibility names continue to resolve the
  same default values through `envctl_engine.config`.
- Backend start-command suggestions remain Python-layout based; unsupported Node-style backend layouts still return an
  empty config default rather than inventing a backend command.
- `config/__init__.py` is reduced further and no longer owns command/default resolution directly.
