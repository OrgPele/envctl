Extract config source discovery ownership

Scope:
- Move `.envctl` and legacy config source discovery, generated-worktree parent config root resolution, explicit config
  path resolution, and `.envctl` text parsing into `config/source_discovery.py`.
- Keep `envctl_engine.config` as the import-compatible facade, including existing private compatibility names used by
  tests or hidden callers.
- Add direct source-discovery tests for envctl text parsing, primary-vs-legacy source precedence, explicit legacy
  prefill, provenance-based parent root resolution, generated tree-shape parent root resolution, and git-boundary
  handling.
- Update the structure guard and Python engine architecture inventory for the new source-discovery owner.

Key behavior changes:
- No intended config semantics changed; `load_config`, `discover_local_config_state`, and private compatibility aliases
  continue to resolve through `envctl_engine.config`.
- Generated worktrees still read the parent repository `.envctl` when provenance or the generated tree path shape
  identifies a parent repo with a primary `.envctl`.
- `config/__init__.py` is reduced to roughly 725 lines and no longer owns config source discovery directly.
