Centralize action artifact naming and persistence

Scope:
- Add a shared artifact-name helper module for filesystem-safe project artifact directories and command report
  filenames.
- Route failed-test summaries and project-action failure reports through the shared artifact naming policy so project
  names with path separators cannot create nested or invalid artifact paths.
- Route action/test summary state persistence through the runtime-context state repository helper instead of direct
  `runtime.state_repository.save_resume_state` access.
- Route project action environment setup through the runtime-context state repository resolver while preserving the
  existing tree-diffs and runtime-root contract.

Key behavior changes:
- Project names such as `Feature/API v2` now produce single safe artifact names like `Feature_API_v2` instead of
  leaking path separators into artifact paths.
- Action support modules now use the runtime-context dependency boundary for state persistence and repository access.
