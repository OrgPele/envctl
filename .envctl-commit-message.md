Route action process execution through runtime context

Scope:
- Route project action command execution and streaming through `resolve_process_runtime`.
- Route migrate action command execution through `resolve_process_runtime`.
- Route worktree git probes and self-destruct helper launch through `resolve_process_runtime`.
- Extend the runtime-context protocol guard so direct action-side process runner access does not return.

Key behavior changes:
- No intended CLI behavior change for valid project, migrate, or worktree actions.
- Action helpers now depend on the shared runtime-context process boundary instead of reaching into runtime internals.
- Missing project-action command configuration is still reported before any process dependency is required.
- The remaining direct process-runner lookup is isolated to `action_runtime_facade.py`, which is an intentional
  compatibility facade.
