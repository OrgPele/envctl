When the current implementation pass finishes, run focused validation with `envctl test-focused --project <current-worktree-name>`, then run `envctl ship --project <current-worktree-name> --json`. If `ship` reports merge conflicts, resolve them using the returned `merge_conflicts` details and rerun validation/ship. `ship` returns immediately with current PR/check state; keep doing useful local work while checks are pending, but before final handoff wait for GitHub status checks to complete and confirm all required checks pass.
