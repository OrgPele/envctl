Run bare `envctl ship` from inside the current worktree/project directory; add `--project <current-worktree-name>` only when operating from elsewhere. `ship` commits, pushes, creates a PR when none exists, reuses or updates the existing PR otherwise, predicts merge conflicts, waits for GitHub checks until they pass, fail, time out, or report no check contexts, and returns current PR status-check state with `pr_created`, `operation_statuses`, `checks_state`, `passed_checks`, `failing_checks`, `pending_checks`, and `checks_error`. When subagents are available, delegate `ship` to a background subagent and keep the main thread on non-overlapping work; do not wait for a successful ship result. The shipping subagent must report commit/push/PR failures, merge conflicts, failed PR checks, pending-check timeout, no-checks-reported status, and actionable review comments immediately; only return to the shipping lane if the subagent reports an issue.
