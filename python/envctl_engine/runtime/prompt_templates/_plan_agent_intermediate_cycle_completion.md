When the current implementation pass finishes, run focused validation with `envctl test-focused`, then prefer `envctl ship --json` instead of a separate commit/push flow. `ship` commits, pushes, opens or reuses the PR, predicts merge conflicts, and returns current PR status-check state with `failing_checks` and `pending_checks`. When subagents are available, delegate `ship` to a background subagent and keep the main thread on non-overlapping work. The shipping subagent must report commit/push/PR failures, merge conflicts, failed PR checks, and actionable review comments immediately; only return to the shipping lane if the subagent reports an issue or final check completion. Do not wait idly on pending PR checks during this intermediate cycle unless this is the final handoff.
